Title: Claude Dispatch or OpenClaw?
Date: 2026-03-21
Slug: dispatch-or-openclaw
Status: published

A few days ago I started setting up OpenClaw on a server in my office. I didn't want to give it access to much, so I wanted to install it in a container with a single filesystem mount, then decide what services to add slowly and intentionally. It took a while to get the permissions sorted out, but with Claude's help (WCH) I was able to get it running.

Then I hit the onboarding and freaked out.

I've never seen such a confusing word salad of services in my life. 

So I stopped.

A few days ago, I heard about Claude Dispatch and decided to give it a try. If OpenClaw is an adolescent personal agentic system, Dispatch seems like an early-twenties agent who has a job and owns a car. It's not as powerful or energetic, but it's also less likely to sell my identity to the highest bidder. From what I've been able to tell so far, Dispatch is a chat-based interface to an instance of Claude Cowork running a computer — in my case, my main Mac.

I haven't quite given Dispatch access to my whole file system but it has more access than I'd ever have given OpenClaw. I'm sure OpenClaw can do a lot more, but until I can think of a compelling application that would drive me to behave so recklessly, I'll stick with Dispatch and hope its features evolve.

I admire Anthropic for how quickly they're iterating their tools, but for the love of God get some sane product management in there. Let's see if I can remember them all... we've got Claude Opus, Sonnet, Haiku, Cowork, Dispatch, Code and plain old Chat. They're not the same family or even similar products. Some are models, some are agentic systems, some are just chat interfaces. The only way it could be more confusing would be if they used numbers. Oh, they do, we've got Opus 4.5, 4.6, Sonnet 4.5, 4.6 and on and on.

Early days.
